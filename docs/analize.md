# Отчет о проведении тестовой игры MusicJam

## Заявленное MVP:
1. Регистрация / вход
2. Создание и настройка игры:
	- Загрузка фрагментов песен
	- Настройка категорий и стоимостей песен
	- Формирование ссылки/кода для подключения
	- Основная информация об игре (автор, название, число команд)
3. Проведение игры:
	- Комната ожидания для игроков
	- **Автоматическое воспроизведение песен при выборе ячейки**
	- **Поле для игроков с категориями**
	- Кнопка для ответа
	- **Автоматический подсчет баллов для каждой команды**
4. Админ-панель для ведущего:
	- Правильные ответы
	- Возможность ручками поправить баллы
	- Табличка и рейтинг

## Состояние проекта на момент первой тестовой игры:
1. Регистрация / вход работают, но ведущего через n минут разлогинивает...
2. Игра создается!
3. В ходе игры работает все, кроме адекватного подсчета баллов
4. Админ-панель работает

## Отчет по метрикам на тестовой игре

Метрика: Доля ведущих, которые после проведения одной игры с помощью сайта, пришли делать новый пак, от числа всех пользователей.

Результат: *За одну игру не измерить, однако можно учесть, что обратная связь, показала желание и ведущего, и игрока поучаствовать в подобном эксперименте снова* 

Метрика: Доля действительно проведенных игр от всех игр, созданных* на сайте

Результат: *Тут не показательно. Пока одну создали - одну провели*

Метрика: Количество операций на сайте, которое делает ведущий ручками во время пака

Результат: *Без нашего сервиса: найти в поиске трек, запустить, остановить, переключиться на табличку, найти команду, посчитать новые баллы, обновить рейтинг, закрасить использованную ячейку, запустить следующий трек. С нашим сервисом: кликнуть по вопросу, принять или отклонить ответ команды, завершить вопрос - всего 3 клика на вопрос.*

Метрика: Количество времени проведенное за созданием пака (сколько времени будет экономиться за этим)

Результат*Результат по тестовой игре: при заранее нарезанных треках создание игры из 55 вопросов заняло +-40 минут с 3-4 ошибками ведущего (загрузил не те песни на вопрос, сервис ему не дал возможности быстро все перепроверить)*

## Результаты за неделю:

1. Исправили пак критичных ошибок:

  - Таблица не приспособлена к большому числу столбцов и строк
  - Никак не работает адаптив в верстке
  - При обновлении страницы может слететь все: рейтинг игроков, добавленные треки, уже проигранные песни и пр.
  - Изменение имени команды почему-то не изменяет, а добавляет новую команду
  - Ведущего (и вроде бы не его одного) в течение игры много раз разлогинивает
  - При выходе из комнаты музыка продолжает играть
  - В рейтинге игроков периодически показывается NaN
  - Идентификация, аутентификация и авторизация не обеспечивают ни приватности, ни безопасности
  - Если игроки нажимают на кнопку после того, как песня доиграла, то появляется NullReferenceException
  - Сокеты слетают. Не всегда у игроков появляется кнопка. Не всегда показывается ответ
  - Нет никакой валидации на стоимости вопросов. 
  - Кнопка "Завершить игру" такая же, как и кнопка "Завершить вопрос"
  - Кнопка "Завершить вопрос" почему-то есть всегда.
  - Отвечающему на вопрос все равно показывается надпись "Кто-то отвечает". Кто-то это ты

2. Попытались договориться о времени проведения следующей игры, но не сошлись в расписании. (Ищем другие варианты)

3. Созвонились в куратором и показали ему промежуточную версию проекта
Даже сыграли на созвоне небольшой пак. Заодно поняли, что не так сложно можем добавить в будущем еще одну функцию: воспроизведение музыки не только у ведущего, но и у всех игроков. Так несложно выйдем и на формат онлайн-мероприятий
